    O telemóvel do motorista não precisa de estar ligado ao Teltonika. O motorista inicia o serviço na app e os pontos do tracker entram na API pelo webhook.

    Checklist produção (Teltonika + Flespi/Traccar + API)

    1. Backend: definir TELTONIKA_WEBHOOK_TOKEN no backend/.env (valor forte, único por ambiente). Reiniciar a API após alterar.
    2. Rede: o servidor da API deve ser acessível a partir do gateway (IP público, domínio, firewall, HTTPS em produção). Testar GET /health a partir de fora da LAN se aplicável.
    3. Teltonika (viatura): APN da operadora correto; servidor aponta para o gateway (Flespi/Traccar), não diretamente para esta app, salvo se o vosso stack receber Codec8 na API.
    4. Frota e chapa: o IMEI deve estar mapeado com frota igual (chapa/matrícula opcional) às usadas quando o motorista inicia o serviço na app — a API associa o tracker ao serviço in_progress por esses campos.
    5. Webhook no gateway: método POST, URL …/integrations/teltonika/events, header Authorization: Bearer <mesmo token do .env>, corpo JSON (ver exemplos abaixo).
    6. Fuso horário: enviar timestamp em ISO UTC (ex. com sufixo Z) sempre que possível.
    7. Teste operacional: com serviço iniciado na app para essa frota, enviar um ponto de teste; confirmar no detalhe do serviço que surgem pontos e que point_source no tracker é aceite.
    8. Segurança: não partilhar o token em email em claro; rotação periódica se suspeita de fuga.
