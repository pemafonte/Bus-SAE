Guia GTFS Real-Time (GTFS-RT)

Este guia mostra como consumir os feeds GTFS-RT da API para posição de viaturas, atualizações de viagem e alertas operacionais.

1) Endpoints disponíveis

Protobuf (produção)

GET /gtfs-rt/vehicle-positions.pb
GET /gtfs-rt/trip-updates.pb
GET /gtfs-rt/service-alerts.pb

JSON (debug e testes)

GET /gtfs-rt/vehicle-positions.json
GET /gtfs-rt/trip-updates.json
GET /gtfs-rt/service-alerts.json

2) Segurança opcional por token

Se definir GTFS_RT_TOKEN no backend/.env, os feeds exigem autenticação por:

  • Authorization: Bearer <GTFS_RT_TOKEN>, ou
  • query param ?token=<GTFS_RT_TOKEN>.
GTFS_RT_TOKEN=defina_um_token_forte

3) O que cada feed entrega

  1. VehiclePositions: latitude/longitude, velocidade, bearing, identificação da viatura, trip e route.
  2. TripUpdates: atraso estimado da viagem e próxima paragem prevista (quando disponível).
  3. ServiceAlerts: alertas operacionais (ex.: transferências pendentes e conflitos registados).

4) Exemplos de consumo rápido

4.1 JSON (debug imediato)

curl "http://localhost:4000/gtfs-rt/vehicle-positions.json"
curl "http://localhost:4000/gtfs-rt/trip-updates.json"
curl "http://localhost:4000/gtfs-rt/service-alerts.json"

4.2 Protobuf com token

curl -H "Authorization: Bearer <GTFS_RT_TOKEN>" \
  "http://localhost:4000/gtfs-rt/vehicle-positions.pb" --output vehicle-positions.pb

5) Checklist de implementação prática

  1. Garantir importação GTFS estático atualizada (/gtfs/import).
  2. Garantir serviços em execução com GPS a entrar em service_points.
  3. Ativar token GTFS-RT em produção (recomendado).
  4. Publicar URLs dos feeds no consumidor externo (app de passageiros / broker).
  5. Monitorizar disponibilidade dos endpoints e latência de atualização.

6) Limitações atuais e evolução recomendada

  • TripUpdates usa atraso estimado com base no estado operacional atual.
  • ServiceAlerts usa alertas operacionais internos já existentes na plataforma.
  • Próxima evolução recomendada: publicar também uma camada de ocupação por viatura e severidade de alertas por linha.
